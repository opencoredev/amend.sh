1. EXECUTION MODEL FOR REMAINING WAVES
------------------------------------------------------------------
Per lane: one builder agent (Fable, async) + one adversarial reviewer agent
(feature-dev:code-reviewer) that tries to BREAK it; orchestrator applies
findings. Wave gate: bun run check-types && bun run lint && bun test &&
bun run build + `bun run smoke` static checks stay green (runtime smoke needs
dev servers). UI lanes additionally: design brief adherence (docs/brand.md
anti-slop list), and screenshot review (agent-browser) when browser validation
is allowed — never tune visuals blind.
Builders must NOT: git add/commit/push, convex dev/deploy (codegen ok), start
dev servers, touch unrelated uncommitted work. Watch for stalls: if a lane's
task output file stops growing for ~30+ min, kill + relaunch with a recovery
brief (check what it left in the tree first; snapshot BEFORE killing).
