# Self-Host, Update, And Doctor Guide

Amend should be useful before any real provider keys are added. A self-hoster can run the product,
seed data, inspect the GitHub-to-customer loop, and then add provider credentials when ready.

## Local Install

```bash
bun install
bun run dev:setup
bun run check
```

The normal development stack uses the repo scripts:

```bash
bun run dev
```

The web app serves the product surfaces, `packages/backend` serves Convex functions, `packages/sdk`
contains the browser/server SDK and embed helper, and `packages/cli` contains the `amend` CLI.

## No-Secret Validation

Use demo mode to verify the product loop without GitHub, AI, Slack, Discord, email, or Stripe keys:

```bash
bun packages/cli/src/index.ts status --demo
bun packages/cli/src/index.ts requests search --demo --query "coding agent"
bun packages/cli/src/index.ts agent run --demo
bun packages/cli/src/index.ts changelog draft --demo --title "Source-linked update"
bun packages/cli/src/index.ts doctor
bun packages/cli/src/index.ts version
```

Demo mode should prove:

- customer requests and votes exist
- GitHub-like source links exist
- the proactive agent produces reviewable recommendations
- changelog/roadmap/outbox surfaces can be inspected
- provider keys are optional for local beta verification

## Provider Keys To Add Later

Set these in Convex or the hosting environment when moving from demo mode to live providers:

- `GITHUB_WEBHOOK_SECRET`
- `GITHUB_APP_ID`
- `GITHUB_APP_SLUG`
- `GITHUB_APP_CLIENT_ID`
- `GITHUB_APP_CLIENT_SECRET`
- `GITHUB_APP_PRIVATE_KEY`
- `AMEND_API_TOKEN`
- `OPENAI_API_KEY` or `ANTHROPIC_API_KEY` when a live AI provider adapter is enabled
- `RESEND_API_KEY`
- `EMAIL_FROM`
- `SLACK_BOT_TOKEN`
- `DISCORD_BOT_TOKEN`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`

Real keys unlock live integrations. They should not be required for local product validation.

## Version And Update Posture

The current package versions are exposed in package manifests and through CLI doctor context:

- root `package.json`
- `packages/backend/package.json`
- `packages/sdk/package.json`
- `packages/cli/package.json`
- `packages/api-spec/package.json`

For production self-hosts, expose the deployed application version from the hosting layer using a
commit SHA or release tag. Prefer these environment variables:

- `AMEND_VERSION`
- `AMEND_COMMIT_SHA`
- `AMEND_UPDATE_CHECK_URL`
- `AMEND_DISABLE_VERSION_CHECK=1`

The running backend exposes this metadata at:

```bash
curl http://127.0.0.1:3211/api/v1/version
```

The CLI can read local package/env metadata, optionally fetch the deployed API metadata, and check
public release metadata only when explicitly requested:

```bash
bun packages/cli/src/index.ts version
bun packages/cli/src/index.ts version --server --check
bun packages/cli/src/index.ts doctor --check
```

The update check is privacy-conscious: it fetches public release metadata only, sends no workspace
data, and can be pointed at a mirror with `AMEND_UPDATE_CHECK_URL` or disabled with
`AMEND_DISABLE_VERSION_CHECK=1`.

Generate an owner token for development with:

```bash
bun packages/cli/src/index.ts token create --limit 32
bun packages/cli/src/index.ts token create --plain
bunx convex env set AMEND_API_TOKEN "paste-generated-token"
```

`token generate` remains as a backwards-compatible alias. Tokens generated by the CLI are local
development credentials; the command does not persist, list, or revoke tokens until the backend has
a scoped token store.

## Upgrade Flow

```bash
git pull --rebase
bun install
bun run dev:setup
bun run check
bun run build
bun packages/cli/src/index.ts doctor
bun packages/cli/src/index.ts version --server --check
```

When production credentials are configured, rerun the provider-specific checks:

```bash
bun test scripts/github-signatures.test.ts
bun test scripts/stripe-signatures.test.ts
bun run readiness
bun run agent-ready
```

## Hosted Versus Open Source

Open-source/self-hosted should include:

- source-linked records
- public portal
- SDK/API/CLI
- embed helper
- deterministic local agent
- BYO provider keys
- local delivery outbox
- docs and doctor checks

Hosted paid plans can add:

- managed proactive agent runtime
- managed email/notification delivery
- more active projects/history
- team permissions and branding controls
- managed domains
- higher volume limits
- priority support

## Doctor Expectations

`amend doctor` should stay fast and safe. It checks local configuration, OpenAPI presence, provider
key posture, and update-check settings without leaking secrets or contacting public repositories.
Pass `--check` when an operator explicitly wants to fetch public release metadata.
